## 2. The cool AI bait offer: The AI Name Check

**Name.** The AI Name Check: see what AI assistants say when a homeowner asks for the best [trade] in your city.

**What it is.** A free, live, self-serve tool at vervemd.com/name-check. The owner types three things they already know: business name, city, and trade. The tool asks the leading AI assistants the same question a homeowner would (who is the best [trade] in [city]) and shows, honestly and on screen, whether the owner is named, whether competitors are named instead, and where they stand. It shows only the assistants that actually returned an answer on that run, and labels them by name. No email required to run it.

**Why it is cool.** It is the moment the abstract becomes personal. Most owners have never once watched a machine answer the exact question their next customer is about to ask, and seeing a competitor's name appear where theirs should be is a gut-punch no slide or stat can deliver. Instead of telling the owner they might be invisible, it shows them being invisible, in their own city, with the competitor names attached. It is built to be defensible for the skeptical owner: each result is labeled by assistant, and where an assistant answers from memory rather than live search we say so plainly. It ties to the one attributed stat: 60% of home service searches now start with an AI tool, not a search engine (BrightLocal 2025).

**How it works.** (1) Owner lands on /name-check and sees one short form: business name, city, trade. (2) They press Run my Name Check; an honest progress state runs while the tool asks each assistant the real homeowner question. (3) The result screen shows, per assistant that returned, whether their business was named and lists the competitor names that came up. If an assistant could not be reached, the screen says so by name rather than dropping it quietly. (4) A plain-English read: AI answers usually surface a short shortlist (often around three names, our estimate), so the question is whether you are one of them. (5) The result hands off to the full free audit, which explains the why and the fix. To start the audit the owner gives name, email, city, and trade, which also starts the bridge.

**CTA.** See if AI names your business. Run the free Name Check at vervemd.com/name-check.

## Before you build: three things to lock (flagged by the stoic + double-checker)

**A. Lock the exact AI-assistant list, once, everywhere.** The copy currently shows "the leading AI assistants" generically and only names them by exception. Your live audit code (`lib/audit/geo.ts`) wires up Claude (default), ChatGPT, Perplexity, and Gemini. Pick the set you will name and use it identically in the Name Check, the /audit copy, and the bridge emails. A skeptical owner who runs the check, then reads an audit naming a different set, notices.

**B. The Name Check must be technically honest, or it backfires.** The stoic caught this: in `geo.ts`, ChatGPT and Gemini use live web grounding, but the Claude call answers from training memory (no web search). Presenting a from-memory answer as "what a homeowner sees live" overstates it. Two fixes when you build: (1) display only the assistants that actually ran and returned, label each by name, and say plainly if one could not be reached; (2) either add web search to the Claude call or label its result distinctly ("based on what Claude knows") versus the search-grounded engines. The copy is already written to support this honesty; the build has to honor it.

**C. Align the audit's named deliverables.** Describe the /audit using its real four deliverables (AI Visibility Score, SEO Health Check, Paid Ads Assessment, Website Conversion Audit). If the Name Check tool shows a competitor list on screen, describe that as a tool-screen feature, not as a separate audit deliverable, so the bait does not redefine the audit it hands off to.
